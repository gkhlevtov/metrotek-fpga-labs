# Лабораторная №3

# Задание №1 LIFO

## Описание

В рамках лабораторной работы был верифицирован модуль LIFO с использованием ООП подхода. Была выстроена многоуровневая тестовая среда, включающая в себя генератор направленных стимулов, драйвер, монитор и скорборд. Основной акцент сделан на поиске критических дефектов в логике защиты от переполнения и опустошения, а также на анализе корректности формирования статусных флагов и временных диаграмм модуля.

## Структура проекта

```
task1/
│── rtl/              # Исходный код
│ └── lifo.sv
├── tb/               # Тестбенч
│ ├── driver.sv       # Драйвер для подачи сигналов в DUT
│ ├── environment.sv  # Контейнер для сборки всех компонентов
│ ├── generator.sv    # Генератор тестовых сценариев
│ ├── lifo_if.sv      # Интерфейс с тактируемыми блоками
│ ├── lifo_pkg.sv     # Пакет с описанием классов и типов
│ ├── lifo_tb.sv      # Верхний уровень тестбенча
│ ├── monitor.sv      # Монитор для захвата данных из DUT
│ ├── run.do          # Скрипт для запуска
│ ├── scoreboard.sv   # Эталонная модель и проверка данных
│ └── transaction.sv  # Описание транзакции
├── report.md         # Отчет о найденных багах
└── README.md
```

## Инструкция по запуску симуляции

1. Перейти в директорию `tb`:
   ```
   cd tb
   ```
2. Запустить ModelSim и выполнить команду:
   ```
   do run.do
   ```

## В ходе работы использовались
